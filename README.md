# Icom705DigitalDecoder — декодер цифровых сигналов с IC-705 на Raspberry Pi

Волкодер ходит по диапазону на IC-705: аналоговый FM слышен сразу, а при появлении
цифрового сигнала (DMR) он на лету декодируется на Raspberry Pi и проигрывается
раскодированным голосом. Декодер — [dsd-fme](https://github.com/lwvmobile/dsd-fme)
(Florida Man Edition), демодуляция — GNU Radio.

## Архитектура

```
IC-705 (USB AF/IF Output = IF, 100% уровень)
  └─ USB Audio CODEC: I/Q-пара (I = левый канал, Q = правый), несущая на +12.14 кГц
       │
       ▼  pw-record (стерео f32; interleaved стерео == complex64)
  GNU Radio (if_demod.py):
       freq_xlating_fir_filter_ccf  — перенос −12140 Гц, ФНЧ ±3.5 кГц
       pwr_squelch_cc (−16 дБ)      — тишина без несущей
       quadrature_demod_cf          — ЧМ-детектор (±1944 Гц → ±1.0)
       float_to_short → tcp_sink    — s16le mono 48k, TCP-сервер :7355
       │
       ▼  TCP (формат SDR++)
  dsd-fme -fs -8 -i tcp -o pulse:<колонки>
       -fs  DMR BS/MS simplex
       -8   Source Audio Monitor: без цифровой синхронизации на выход идёт
            исходный (аналоговый) звук — само переключение аналог/цифра
```

## Ключевые факты (выяснены экспериментально)

- **IF-выход IC-705 — квадратурный.** Это не моно 12-кГц IF: I и Q разнесены по
  стерео-каналам (сдвиг 90°, корреляция ~0). Обработка одного канала как моно
  складывает спектр пополам — зеркальный канал даёт неустранимый пол битовых
  ошибок («кваканье» вокодера) и ложную «инверсию» полярности (-xr).
  Комплексный тракт: ошибок голоса в 33 раза меньше на эталонной записи.
- **Несущая на +12140 Гц** (не ровно 12000) — измерено по гистограмме уровней 4FSK.
- **Мост к PipeWire — только pw-record + TCP.** Пара pw-record/pw-play — два
  независимых клиента с разными тактовыми доменами: поток рвётся, декодер теряет
  синхронизацию каждые 1–3 голосовых кадра. TCP-потребитель темпа не навязывает.
- **dsd-fme обязательно собирать с оптимизацией**
  (`cmake -DCMAKE_BUILD_TYPE=Release -DCMAKE_C_FLAGS='-mcpu=cortex-a53'`):
  дефолтная сборка (-O0) на Pi 3 не успевает за реальным временем (100% CPU,
  растущие TCP-очереди, артефакты). После пересборки — 60–85% CPU с запасом.
  Запускать с `renice -15`.
- **Порог сквелча −16 дБ**: сигнал в канале −10.5 дБ, шум −22 дБ (окно 11.5 дБ).

## Файлы

| Файл | Назначение |
|------|-----------|
| `if_demod.py`            | живой демодулятор: stdin (I/Q f32) → TCP :7355 |
| `run_pipeline.sh`        | запуск: pw-record → if_demod.py |
| `if_demod_offline_iq.py` | оффлайн-вариант (файл→файл) для итераций на эталонной записи |
| `if_demod_offline.py`    | устаревший моно-вариант (история: до открытия I/Q) |
| `sweep_params.sh`        | перебор центра/полосы с метриками dsd-fme (моно-эпоха) |

## Запуск на Pi (10.1.1.44, radiodecoder)

```bash
nohup bash run_pipeline.sh > pipeline.log 2>&1 &
nohup dsd-fme -fs -8 -i tcp \
  -o pulse:alsa_output.platform-3f00b840.mailbox.stereo-fallback \
  -J ~/dsd_events.log > ~/dsd.log 2>&1 &
sudo renice -15 $(pgrep -x dsd-fme)
```

Настройки IC-705: `Connectors → USB AF/IF Output → AF/IF = IF`, уровень 100%.
