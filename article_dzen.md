# Научил Icom IC-705 декодировать DMR, D-STAR и P25 одновременно — помогла Raspberry Pi и один вечер экспериментов

*[Место для обложки: фото IC-705 рядом с Raspberry Pi, на экране ноутбука — веб-дашборд с водопадом]*

Крутишь валкодер по диапазону, слушаешь эфир — и вдруг вместо голоса из динамика лезет цифровое «жужжание». DMR, D-STAR, P25 — цифровых видов связи в эфире всё больше, а мой IC-705 из коробки понимает только D-STAR. Покупать отдельную рацию под каждый протокол? Нет уж.

Я сделал иначе: маленькая Raspberry Pi слушает трансивер по USB и на лету декодирует цифру. Аналоговый FM играет как обычно, без задержки. А как только на частоте появляется цифровой сигнал — из колонок идёт уже раскодированный голос. Плюс веб-страничка с водопадом, журналом услышанных операторов и кнопками диапазонов — прямо как экран самого трансивера, только в браузере.

Весь проект — вечер работы и ноль дополнительного железа, если Raspberry Pi у вас уже валяется в ящике. Рассказываю, что получилось и как повторить.

## Что умеет система

- **Декодирует DMR, D-STAR и P25** одним процессом — что поймает, то и раскодирует. Слышно позывной, ID радиостанции, таймслот, Color Code, разговорную группу.
- **Аналог проходит сразу**: пока цифры нет, слушаешь обычный FM. Сквелч убирает шум пустой частоты.
- **Веб-дашборд**: живой водопад (спектр берётся с самого IC-705 по CAT-интерфейсу), S-метр, частота, статус декодера. Клик по водопаду — радио перестраивается на эту частоту. Кнопки диапазонов работают как родные кнопки Icom — возвращают на последнюю частоту диапазона.
- **Журнал операторов**: каждая услышанная передача пишется в базу — время, частота, вид связи, позывной, ID.
- **Всё поднимается само** после перезагрузки и переживает пропадание USB.

## Как это устроено (в двух словах)

Главная находка проекта: если в меню IC-705 переключить USB-аудиовыход из режима AF в режим **IF**, трансивер отдаёт по USB не готовый звук, а «сырую» промежуточную частоту 12 кГц. Причём — это нигде толком не описано — в стерео-каналах прячется квадратурная пара I/Q, как у настоящего SDR-приёмника.

Дальше цепочка такая:

1. **GNU Radio** (среда для программной обработки радиосигналов) принимает этот I/Q-поток и по-честному демодулирует ЧМ — чище, чем это делает аудиотракт самого трансивера.
2. **DSD-FME** (Digital Speech Decoder, свободный декодер цифровой речи) получает демодулированный сигнал и разбирает протоколы: DMR, D-STAR, P25, YSF. Голос синтезирует библиотека mbelib.
3. Маленький **веб-сервер на Python** показывает всё в браузере и управляет радио по CAT-протоколу CI-V.

Радио при этом работает в обычном FM-режиме — просто крутишь частоту, а Pi сама разбирается, аналог там или цифра.

## Установка одним скриптом

Понадобится Raspberry Pi 3 или новее с чистой Raspberry Pi OS Lite и IC-705, подключённый по USB. Три команды в терминале:

```
sudo apt-get install -y git
git clone https://github.com/sinty/Icom705DigitalDecoder.git
cd Icom705DigitalDecoder && bash install.sh
```

Скрипт сам ставит пакеты, собирает декодер с нужной оптимизацией, настраивает автозапуск и запускает сервисы. Через 15–20 минут (большая часть — компиляция) дашборд доступен по адресу `http://<ip-малинки>:8080/`.

На самом трансивере надо один раз настроить меню: `SET → Connectors → USB AF/IF Output → AF/IF = IF`, уровень 100%. И проверить, что `CI-V USB Baud Rate = Auto`, `CI-V Address = A4h` (заводские значения).

## Установка руками — для тех, кто хочет понимать каждый шаг

**1. Пакеты.** Аудиостек PipeWire, GNU Radio и инструменты сборки:

```
sudo apt-get install -y pipewire pipewire-pulse wireplumber \
  pipewire-audio-client-libraries pulseaudio-utils \
  git cmake build-essential pkg-config \
  libpulse-dev libasound2-dev libncurses-dev libusb-1.0-0-dev \
  libsndfile1-dev libfftw3-dev liblapack-dev libcodec2-dev socat \
  gnuradio python3-numpy python3-serial
```

**2. Разрешить пользователю работать без входа в систему** (иначе звук умрёт при отключении SSH) и дать доступ к CAT-порту:

```
sudo loginctl enable-linger $USER
sudo usermod -aG dialout $USER
```

**3. Собрать mbelib** — библиотеку-вокодер, которая превращает цифровые кадры в голос:

```
git clone https://github.com/lwvmobile/mbelib ~/mbelib
cd ~/mbelib && git checkout ambe_tones
mkdir build && cd build
cmake -DCMAKE_BUILD_TYPE=Release -DCMAKE_C_FLAGS='-mcpu=cortex-a53' ..
make -j4 && sudo make install && sudo ldconfig
```

**4. Собрать DSD-FME.** Внимание, важный момент: **обязательно с флагом Release**. Сборка «по умолчанию» идёт без оптимизации, и на Raspberry Pi 3 декодер не успевает за реальным временем — звук превращается в кашу. Я на этом потерял пару часов:

```
git clone https://github.com/lwvmobile/dsd-fme ~/dsd-fme
cd ~/dsd-fme && mkdir build && cd build
cmake -DCMAKE_BUILD_TYPE=Release \
      -DCMAKE_C_FLAGS='-mcpu=cortex-a53' -DCMAKE_CXX_FLAGS='-mcpu=cortex-a53' ..
make -j4 && sudo make install && sudo ldconfig
```

**5. Файлы проекта** — демодулятор, веб-дашборд и вспомогательные модули копируются в домашний каталог:

```
cd ~/Icom705DigitalDecoder
cp if_demod.py run_pipeline.sh web.py civ.py scope.py config.py ~/
```

Если у вас не Pi 3, загляните в `config.py`: имя звукового выхода (`speaker_sink`) на Pi 4/5 отличается — посмотреть своё можно командой `pactl list short sinks`.

**6. Автозапуск.** Три systemd-юнита из папки `deploy/` — демодулятор, декодер и веб — ставятся в систему:

```
sudo cp deploy/icom-*.service /etc/systemd/system/
sudo cp deploy/logrotate-icom /etc/logrotate.d/icom
sudo systemctl daemon-reload
sudo systemctl enable --now icom-demod icom-dsd icom-web
```

В юнитах прописан пользователь `sinty` и его домашний каталог — замените на свои (или доверьте это скрипту `install.sh`, он делает то же самое автоматически).

Готово: `http://<ip>:8080/` — водопад, S-метр, журнал.

## Грабли, на которые я уже наступил за вас

- **IF-выход IC-705 — это I/Q, а не моно-сигнал.** Если обрабатывать только один канал, зеркальный канал заворачивается в полосу сигнала и вокодер «хрипит». Понять это помогла спектрограмма — сигнал в комплексном спектре сидел несимметрично.
- **DSD-FME без оптимизации не тянет на Pi 3.** 100% загрузки ядра, растущие очереди, артефакты. Флаг `-DCMAKE_BUILD_TYPE=Release` решает всё.
- **Звук между процессами — только через TCP.** Попытка склеить демодулятор и декодер через виртуальную звуковую карту провалилась: у двух аудиоклиентов разные «часы», поток рвётся каждые пару секунд. У DSD-FME есть родной TCP-вход — им и надо пользоваться.
- **IC-705 заряжается по USB и может перегрузить порт малинки.** У меня Pi однажды вообще не загрузилась из-за этого. Лечится отключением зарядки в меню радио или USB-хабом с внешним питанием.

## Итог

IC-705 + Raspberry Pi + свободный софт = универсальный приёмник цифровой связи с веб-интерфейсом. Ходишь по диапазону валкодером (или кликаешь по водопаду в браузере), аналог слышен сразу, цифра декодируется сама, все услышанные операторы — в журнале.
